## 📌 Instructions

A fast-growing infrastructure technology company called LiftGrid Systems builds intelligent elevator control platforms for large commercial buildings across India.

Their software is used in corporate towers, malls, airports, hospitals, and high-rise residential complexes where multiple elevators operate together across many floors.

Unlike standalone lifts, these buildings contain multiple elevator shafts and elevators grouped into zones, handling thousands of passenger requests daily. The system must efficiently manage elevator assignments, floor requests, maintenance tracking, and ride history.

Each building can contain multiple elevator shafts. Each shaft contains one elevator. Each elevator operates across a defined set of floors and responds to ride requests generated by users.

The platform should support:

* Multiple buildings
* Multiple elevators inside each building
* Floor-level request tracking
* Ride allocation to elevators
* Elevator status monitoring
* Maintenance tracking
* Usage history logging

This is a **multi-building infrastructure management system** where real-time requests, elevator allocation, and operational tracking must be modeled properly.

## 🤔 Things to Think About

* One building contains multiple floors
* One building contains multiple elevators
* Elevators belong to buildings
* Each elevator may serve multiple floors
* A floor may be served by multiple elevators
* Users generate requests from floors
* Requests are assigned to elevators
* One request results in one ride allocation
* Elevators complete multiple rides daily
* Maintenance should not overwrite historical data
* Elevator status should be tracked over time (not stored statically)
* Avoid storing dynamic ride data inside the elevator entity
* Use junction tables where many-to-many relationships exist
